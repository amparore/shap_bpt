# Contributing to ShapBPT

Thank you for your interest in contributing to **ShapBPT**!
We welcome contributions from researchers, engineers, and practitioners working on explainable AI and computer vision.

---

## 🚀 Ways to Contribute

You can contribute in several ways:

* 🐛 Reporting bugs
* 💡 Suggesting new features or improvements
* 📚 Improving documentation
* 🧪 Adding examples or notebooks
* ⚙️ Improving performance or code quality
* 🧠 Contributing new XAI methods or extensions

---

## 🛠 Getting Started

### 1. Fork and clone the repository

```bash
git clone https://github.com/<repo-owner>/shap_bpt.git
cd shap_bpt
```

### 2. Create a new branch

```bash
git checkout -b feature/your-feature-name
```

### 3. Install dependencies

```bash
pip install -e .
```

For documentation:

```bash
pip install -r docs/requirements.txt
```

---

## 🧪 Development Guidelines

* Follow clean and readable coding practices
* Add comments where necessary
* Prefer modular and reusable functions
* Keep changes focused and minimal

---

## 📚 Documentation Contributions

We highly encourage improving documentation:

* Fix typos or unclear explanations
* Improve tutorials and notebooks
* Add new example use cases

To build docs locally:

```bash
cd docs
make html
```

---

## 🧠 Contributing to XAI Methods

If you are adding new explainability methods or improvements:

* Ensure consistency with the existing API
* Provide a minimal working example
* Include visualization support if applicable
* Clearly document assumptions and limitations

---

## 🧪 Testing

Before submitting a pull request:

* Ensure your code runs without errors
* Test on at least one example notebook
* Verify compatibility with existing API

---

## 📦 Pull Request Process

1. Ensure your branch is up to date
2. Provide a clear description of your changes
3. Reference related issues (if any)
4. Include screenshots or outputs for visualization changes
5. Keep pull requests focused and concise

---

## 🐛 Reporting Issues

When reporting a bug, please include:

* Description of the issue
* Steps to reproduce
* Expected vs actual behavior
* Environment details (OS, Python version, dependencies)
